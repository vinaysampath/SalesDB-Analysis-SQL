# E-Commerce Sales Data Portfolio: Cleaning & Exploratory Data Analysis (SQL)

## Project Overview
This repository contains a comprehensive SQL-based portfolio project focused on maintaining data integrity and uncovering actionable business insights from an e-commerce database (`SalesDB`). 

The project is split into two foundational phases:
1. **Data Cleaning & Quality Assurance:** Eliminating redundant records and resolving missing/null operational data.
2. **Exploratory Data Analysis (EDA):** Writing optimized queries to analyze customer behavior, product performance, regional distribution, and operational efficiency.

---

## Database Architecture
The queries interact with the `Sales` schema, utilizing the following core tables:
* **`sales.Orders` / `sales.OrdersArchive2`**: Operational transaction ledgers.
* **`sales.Customers`**: Demographic and profile information for clients.
* **`sales.Employees`**: Internal staff data, department assignments, and performance tracking.
* **`sales.Products`**: Merchandising catalog containing categories and pricing structures.
